MSPM0(CCS)重定向printf

在重定向printf的时候往往是直接重写printf调用的fpuc函数，就像这样：

但是在CCS中可能会出现这样的错误：error #10056: symbol "fputc" redefined: first defined in "./main-blinky.o"; redefined in "C:\ti\ccs2011\ccs\tools\compiler\ti-cgt-armllvm_4.0.2.LTS\lib\armv6m-ti-none-eabi/c/libc.a&lt;fputc.c.obj&gt;"

其原因是TI默认的C/C++库的fpuc没有设置weak属性，像这样：__attribute__((weak)) fputc(int ch, FILE *stream)
导致自己编写的fputc不能覆盖原先的定义，所以最好的方法是：重新写一个printf

如果在执行 vsnprintf 的过程中程序卡死（进了错误中断），是因为它访问了不该访问的内存（栈内存太小了），常见的作法就是扩大栈内存的大小，在 CCS 中的项目文件夹右键选择属性(properties)，在arm linker的设置下面设置栈内存（Set C system stack size）的大小
